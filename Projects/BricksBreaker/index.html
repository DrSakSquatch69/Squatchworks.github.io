Bricks Breaker

C++ Windows API 2D Physics OOP Solo Project
Bricks Breaker Gameplay

About

Bricks Breaker is a faithful recreation of the classic brick-breaking formula. Built using the Windows API for rendering, the project served as a deep dive into 2D collision physics, paddle movement logic, and level state management.

The codebase emphasizes clean OOP principles and efficient memory management within a constrained environment, with no external game engine or framework.

My Contributions

  • Designed and implemented the core ball physics system including velocity, angle reflection, and edge-case collision handling
  • Built the paddle movement and input handling system using Windows API keyboard events
  • Created the brick grid system with configurable layouts and destruction logic
  • Implemented game state management — start screen, active play, win/loss conditions, and restart flow
  • Applied OOP design principles with clean class separation for Ball, Paddle, Brick, and Game Manager entities

Features

  • Ball Physics — Angle-based reflection off paddle, walls, and bricks with accurate collision response
  • Paddle Control — Smooth keyboard-driven paddle movement with boundary clamping
  • Brick Grid — Multi-row brick layout with per-brick destruction and score tracking
  • Win / Loss States — Full game loop with clear state transitions and restart support
  • Windows API Rendering — Direct GDI rendering without any engine abstraction
  • OOP Architecture — Clean class design separating game entities and responsibilities
