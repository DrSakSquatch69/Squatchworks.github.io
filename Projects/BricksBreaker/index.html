About

Bricks Breaker is a classic arcade-style brick breaking game built entirely from scratch in C++ using a console-based renderer. Every system — the ball physics, brick grid, paddle input, and collision detection — was implemented manually without a game engine.

This was an early foundational project focused on core OOP design patterns and real-time game loop architecture. No libraries, no engine — just raw C++ and the Windows console API.

Features

  • Real-Time Game Loop — continuous update/render cycle managing all game state
  • Ball Physics — velocity-based movement with angle reflection off walls, paddle, and bricks
  • Custom Collision Detection — AABB collision between ball, paddle, bricks, and boundaries
  • Destructible Brick Grid — hit detection and brick removal on contact
  • Paddle Control — keyboard-driven movement with boundary clamping
  • Win/Loss States — game over on ball loss, win condition on brick clear
  • Console Renderer — Windows console API with cursor positioning and character rendering

Architecture

  • Game — core game loop, state management, collision orchestration
  • Ball — position, velocity, movement, reflection logic
  • Box — brick and paddle entity with active state tracking
  • BaseObject — shared base class for position, size, and render properties
  • Console — Windows console output system
