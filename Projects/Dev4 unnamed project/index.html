About

This is the individual prerequisite built before Arachnivasion. Every team member built this solo first to earn deep ownership of the shared engine architecture. Each student independently implemented the same Vulkan/EnTT engine base before the team merged their work into the collaborative Arachnivasion project.

Building this solo first meant every team member had deep ownership of the underlying architecture. The codebase demonstrates direct Vulkan API usage, modular ECS design, and 3D game loop fundamentals without any engine abstraction layer.

My Contributions

  • Built the complete Vulkan rendering pipeline from scratch — buffer management, GPU instance updates, and draw pipeline
  • Implemented the EnTT ECS architecture with CCL (Connect Component Logic) signal-driven component updates
  • Created the player system with keyboard-driven first-person movement and configurable speed
  • Developed enemy entity spawning and management via SpawnHelpers
  • Built the Game Manager for centralized state tracking of player/enemy visibility and entity collections
  • Implemented the Model Manager for 3D model loading and entity creation from model assets
  • Created the Level Loading system using a custom level component for environment geometry

Features

  • Custom Vulkan Renderer — Direct API rendering with buffer management, GPU instance updates, and draw pipeline
  • ECS Architecture — EnTT-based with CCL signal-driven component updates
  • Player System — Keyboard-driven first-person movement with configurable speed
  • Enemy Entities — Enemy spawning and management via SpawnHelpers
  • Game Manager — Centralized state tracking for player/enemy entities
  • Model Manager — 3D model loading and entity creation from model assets
  • Level Loading — Custom level component system for environment geometry
